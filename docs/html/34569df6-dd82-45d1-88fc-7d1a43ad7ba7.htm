Sharp IMG Viewer Help

This topic contains answers to Frequently Asked Questions

The following are some common problems and solutions along with some known issues with the Sharp IMG Viewer 2008 that you may encounter.

Frequently Asked Questions (FAQ)

Please read questions and answers:

  • The default installations of Windows Server 2003 and Windows Server 2008 do not enable the Windows Image Acquisition (WIA) service, which is required to be able to acquire images on digital cameras, scanners, or Web cameras. You can enable WIA as follows:

    Windows Server 2003

    1. Click the Start button and open the Programs menu.

    2. In the Administrative Tools sub-menu, click Services.

    3. Locate "Windows Image Acquisition", right-click it and choose Properties.

    4. Change the Startup type to "Automatic".

    5. Click OK.

    6. Right click "Windows Image Acquisition" again, then click Start.

    Windows Server 2008

    • The WIA service is not installed during the default Windows Server 2008 installation. Before you can enable WIA, it first needs to be installed:

      1. Click the Start button, open the Administrative Tools menu, and click Server Manager.

      2. Click Features to load the Features Summary, then click Add features.

      3. Check the Desktop Experience check box.

      4. Click Next, then click Install.

      5. Restart your computer.

    • Now you can enable WIA as follows:

      1. Click the Start button, open the Administrative Tools sub-menu, and click Component Services.

      2. In the Component Services dialog, click Services (Local) to load the list of services.

      3. Locate and right-click "Windows Image Acquisition", then click Properties.

      4. In the General tab, set the Startup type to "Automatic".

      5. Click OK.

      6. Right click "Windows Image Acquisition" again, then click Start.

  • Opening AVI movies produced by digital cameras frames aren't displayed. Processing these frames message is shown: "Video for Windows compatible codec is not installed". To display frames you should install decoder as described:

    Video for Windows compatible codec installation

    • Download and install ffdshow VFW codec.

    • Launch ffdshow VFW configuration and enable MJPG decoder. You can also enable other supported decoders if needed.

  • To open and view PDF files you have to install the corresponding Microsoft Visual Studio C++ Redistributable Package. You can perform Sharp IMG Viewer update for this purpose.

    Computers with the Microsoft .NET Framework 4.* installed

    • Download and install Microsoft Visual Studio 2010 SP1 C++ Redistributable Package for your platform.

    Computers with the Microsoft .NET Framework 2.0/3.* installed

    • Download and install Microsoft Visual Studio 2008 SP1 C++ Redistributable Package for your platform.

  • Opening images from the specified location is failed. Processing these images message is shown: "The underlying connection was closed: An unexpected error occurred on a send.". To open images you should configure the .NET Framework to support security protocols TLS 1.2 and later as described:

    Configure for strong cryptography

    • For 32-bit applications that are running on 32-bit OSs and for 64-bit applications that are running on 64-bit OSs, update the following sub-key values:

      Registry
      Copy
      [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\.NETFramework\v2.0.50727]
      "SystemDefaultTlsVersions" = dword:00000001
      "SchUseStrongCrypto" = dword:00000001
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\.NETFramework\v4.0.30319]
      "SystemDefaultTlsVersions" = dword:00000001
      "SchUseStrongCrypto" = dword:00000001

    • For 32-bit applications that are running on 64-bit OSs, update the following Sukey values:

      Registry
      Copy
      [HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\.NETFramework\v2.0.50727]
      "SystemDefaultTlsVersions" = dword:00000001
      "SchUseStrongCrypto" = dword:00000001
[HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Microsoft\.NETFramework\v4.0.30319]
      "SystemDefaultTlsVersions" = dword:00000001
      "SchUseStrongCrypto" = dword:00000001
